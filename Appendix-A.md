# Appendix A. Screening and Extraction Fields

**Table 16. Screening and Extraction Fields**

| Field | Operational note |
|---|---|
| Record ID | Unique screening identifier |
| Database source | Scopus, PubMed, IEEE Xplore, ScienceDirect, Web of Science, or ACM |
| Decision at the title/abstract stage | Include, exclude, or unclear |
| Reason for exclusion | Use a controlled vocabulary to enable transparent PRISMA accounting |
| Healthcare domain | Oncology, neurology, diagnostics, documentation, chronic disease, operations, etc. |
| Population/user | Patients, clinicians, health-system staff, or mixed |
| Agentic mechanism | Single agent, multi-agent, tool use, retrieval, memory, self-correction, orchestration |
| Study design | Simulation, retrospective validation, implementation, randomized trial, etc. |
| Comparator | Clinician-only, single LLM, conventional model, none |
| Primary outcomes | Accuracy, safety, usability, completion rate, traceability, etc. |
| Risk-of-bias tool | RoB 2, ROBINS-I, QUADAS-2, plus CASP narrative appraisal |
| Synthesis group | Domain cluster used in narrative synthesis |
